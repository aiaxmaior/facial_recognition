Network Architecture

Headless NAT Gateway

Configuring the Jetson Orin Nano as a DHCP server and internet gateway for isolated Ethernet peripherals.

⚙️ Core Networking Logic

Shared Connection (ICS)

Using the Jetson as a bridge to share Wi-Fi internet with the Ethernet port (eno1).

Local DHCP Server

The Jetson automatically assigns IP addresses (10.42.0.x) to devices connected to the switch.

NAT Forwarding

Translating internal camera requests so they can reach the external internet via the Jetson.

Deployment Commands

Run these commands in order to build the gateway.

bash — qdrive@orin-dev

Phase 1: Internet Connection Sharing

Initialize the Ethernet interface as the gateway for the local subnet.

sudo nmcli con add type ethernet con-name "InternetShare" ifname eno1 ipv4.method shared
sudo nmcli con up InternetShare
sudo sysctl -w net.ipv4.ip_forward=1
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf

Phase 2: NAT Rules & Docker Bypass

Force the firewall to allow DHCP traffic and forward RTSP ports, bypassing Docker's DROP policies.

sudo iptables -I INPUT 1 -i eno1 -p udp --dport 67:68 --sport 67:68 -j ACCEPT
sudo iptables -t nat -F
sudo iptables -t nat -A PREROUTING -p tcp --dport 554 -j DNAT --to-destination 10.42.0.159:554
sudo iptables -I FORWARD 1 -p tcp -d 10.42.0.159 --dport 554 -j ACCEPT
sudo iptables -t nat -A POSTROUTING -j MASQUERADE

Phase 3: System Hardening

Ensure the DHCP server and NAT rules persist across system reboots.

sudo apt-get install iptables-persistent
sudo netfilter-persistent save

Verification: Check the lease file to confirm the camera is connected:
cat /var/lib/NetworkManager/dnsmasq-eno1.leases

Network Overhead: Direct vs NAT

NAT adds negligible (~1%) latency for local RTSP feeds.

Infrastructure Logic

1

Jetson initializes dnsmasq on eno1.

2

Camera broadcasts DHCP DISCOVER.

3

Jetson offers IP 10.42.0.x.

4

IPtables forwards incoming RTSP requests from Wi-Fi to Ethernet.
