2. Configuration Workflow

Follow these phases sequentially to configure the network and initialize the camera.

The Jetson Handshake

The Jetson must act as a router. We configure eno1 to share its internet connection and create a local subnet (10.42.0.x).

Critical: Fix Docker Conflicts

Docker firewall rules often block DHCP. You must run the iptables flush commands.

sudo nmcli con add type ethernet con-name "InternetShare" ifname eno1 ipv4.method shared
sudo nmcli con up InternetShare
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
sudo iptables -I INPUT 1 -i eno1 -p udp --dport 67:68 --sport 67:68 -j ACCEPT
sudo iptables -t nat -F

The "Laptop Bridge"

New Reolink cameras have locked ports. We use a laptop to "punch through" the initial setup wizard.

  • 1 Connect laptop to the same PoE switch.
  • 2 Set laptop Ethernet IP manually to 10.42.0.50.
  • 3 Open Reolink Client, find the camera, and set a password.
  • 4
    CRITICAL: Go to Network > Advanced > Port Settings.
    Toggle RTSP & HTTP to ON.

Clean Feed for AI

Why Sub-Stream for AI?

Port Forwarding & Persistence

Expose the RTSP stream to your main network via the Jetson's Wi-Fi IP and ensure rules survive reboots.

sudo iptables -t nat -A PREROUTING -p tcp --dport 554 -j DNAT --to-destination 10.42.0.159:554
sudo iptables -I FORWARD 1 -p tcp -d 10.42.0.159 --dport 554 -j ACCEPT
sudo iptables -t nat -A POSTROUTING -j MASQUERADE
sudo apt-get install iptables-persistent
sudo netfilter-persistent save
